Generate tick level data from source_folder. The tick level data is generated by FutureTradingUtils.generate_concise_data. :param raw_data_folder: string, the folder storing raw trading data. In the raw data folder, there are

sub-folders for each trading day. The sub-folder names yyyymmdd.
Parameters:
  • tick_data_folder – string, the folder to store generated tick level data. If the tick data folder does not exist, create the folder. In tick_data_folder, data for an instrument on a trading day is stored in a file instrument_id/yyyymmdd.txt if output_text is True, otherwise, the output file is named yyyymmdd.dat. Use ctxalgolib.ohlc.ohlc_reader_writer.OhlcCsvWriter for writing text files, and OhlcBinaryFileWriter for writing binary files.
  • with_open_interests – boolean, if True, include open interests in the generated data.
  • order_book_level – int, specify to which order book we include the data in the result. 0 means no order book. 1 ~ 5 means the upper bound of the order book level to include.
  • dominant – boolean, if True, include the attribute for dominant contract switching.
  • start_date – date, the start date (inclusive) to generate data. If None, start from the first available trading day.
  • end_date – date, the end date (inclusive) to generate data. If None, end at the last available trading day.
  • add_missing_ticks – bool, If True, add missing ticks by duplicating the last tick. Add one tick per second until market periods ends.
  • output_text – boolean, if True use OhlcCsvWriter to store generated data, otherwise use OhlcBinaryFileWriter to store generated data. When using OhlcCsvWriter, set include_date_and_time to False, set include_instrument_id to False.

Generate ohlc data :param raw_data_folder: string, the folder to store raw stock 3s trading data.

The folder should contain one or more sub-folders; each sub-folder is named yyyymmdd. A sub-folder contains stock 3s trading data: in the sub-folder, there will be a list of files, each file is named stock_id_yyyymmdd.csv. For example, 600338_20140913.csv. Note that the stock ids do not have sh or sz prefixes, but in the generated ohlc files or in the generated file content, sh and sz prefixes are needed. Use SecurityLoader.exchange_from_stock_id_number to get the prefix from stock id number.
Parameters:
  • ohlc_folders – dict from Periodicity to string. Keys are the ohlc periodicities, and values are folders to store the generated ohlc files. In the folder, each stock is generated as a file. Use ctxalgolib.ohlc.ohlc_reader_writer.OhlcCsvWriter for writing text files, and OhlcBinaryFileWriter for writing binary files. The file name should be stock_id_period.csv. For example, sh600338_5m. Text file should have .csv extension, binary file should have .dat extension.
  • use_mid_ask_bid – boolean, if True, use ask_price1 and bid_price1 to generate ohlc bars.
  • with_open_interest – boolean, if True, generate open interest information. For stocks, open_interest is always equals to 0 because there is no such data from the underlying data source.
  • order_book_level – int, specify to which order book we include the data in the result. 0 means no order book. 1 ~ 5 means the upper bound of the order book level to include.
  • dominant – boolean, if True, include the attribute for dominant contract switching. Since stocks do not have the dominant concept. If this True, always set the attribute to 0.
  • start_date – date, the start date (inclusive) to generate data. If None, start from the first available trading day.
  • end_date – date, the end date (inclusive) to generate data. If None, end at the last available trading day.
  • add_missing_bars – boolean, if True, add missing bars to the end of each trading day’s data.
  • include_instrument_id – boolean, should instrument id be included in the generated ohlc file?
  • include_extra_date_and_time – boolean, If True, include two separate date and time column in the generated ohlc file.
  • output_text – boolean, if True use OhlcCsvWriter to store generated data, otherwise use OhlcBinaryFileWriter to store generated data. When using OhlcCsvWriter, set include_date_and_time to False, set include_instrument_id to False.
  • accepted_instrument_ids – [string], the list of instrument IDs to process. If None, process all seen instruments.
  • l1_statistics – boolean, if True, include the l1 statistics information
  • header – boolean, if True, include the header row

In the raw data folder, there are

sub-folders for each trading day. The sub-folder names yyyymmdd.
Parameters:
  • tick_data_folder – string, the folder to store generated tick level data. If the tick data folder does not exist, create the folder. In the target_folder, data for a trading day is stored in a file yyyymmdd.txt if output_text is True, otherwise, the output file is named yyyymmdd.dat. Use ctxalgolib.ohlc.ohlc_reader_writer.OhlcCsvWriter for writing text files, and OhlcBinaryFileWriter for writing binary files.
  • with_open_interests – boolean, if True, include open interests in the generated data.
  • order_book_level – int, specify to which order book we include the data in the result. 0 means no order book. 1 ~ 5 means the upper bound of the order book level to include.
  • dominant – boolean, if True, include the attribute for dominant contract switching.
  • choose_main –

    boolean, if True, use volume to choose the main instrument. If False, generate separate files for each instrument under the same instrument id prefix. So, if choose_main is True, tick_data_folder should have the following structure: tick_data_folder/instrument_id_prefix main_contract_postfix/yyyymmdd.csv. For example, tick_data_folder/IF99/yyyymmdd.csv. If choose_main is False, the tick_data_folder should have the following structure:

    tick_data_folder/instrument_id/yyyymmdd.csv.

    The extension can be .csv in case of text output file or .dat in case of binary output file.

  • start_date – date, the start date (inclusive) to generate data. If None, start from the first available trading day.
  • end_date – date, the end date (inclusive) to generate data. If None, end at the last available trading day.
  • add_missing_ticks – bool, If True, add missing ticks by duplicating the last tick. Add one tick per second until market periods ends.
  • output_text – boolean, if True use OhlcCsvWriter to store generated data, otherwise use OhlcBinaryFileWriter to store generated data. When using OhlcCsvWriter, set include_date_and_time to False, set include_instrument_id to False.
  • dominant_level – int, the level of dominant, 0 means the current dominant, 1 means the next dominant, and so on.
  • profits – boolean, if True, generate the profits series.

  • choose_main – boolean, if True, use volume to choose the main instrument. If False, generate separate files for each instrument under the same instrument id prefix. To decide which instrument id to put in ohlc file: 1. if instrument_id_to_include is not None, use that id, otherwise, 2. if choose_main is False, use the full instrument id, such as IF1503. 3. if choose_main is True, use instrument_id_prefix. This parameter is only used with output_text is True.
  • instrument_id_to_include – string, the instrument id used to put into the generated ohlc file. See the comment for include_instrument_id for more details. This parameter is only used with output_text is True.
  • include_extra_date_and_time – boolean, If True, include two separate date and time column in the generated ohlc file. This parameter is only used with output_text is True.
