Type check state machine given by the DOT specification. :param dot: string, the DOT specification for a state machine. :return: [CodeError], list of type checking errors. If no error is detected, return an empty list.

class ctxalgolib.trading_utils.state_machine_type_checking.BasicStateMachineTypeChecker¶

Bases: ctxalgolib.trading_utils.state_machine_type_checking.AbstractStateMachineTypeChecker

This type checker checks the following errors:. 1. The state machine must have a start node.

Error code: 1.
  1. A node can only contain the following properties: start, immediate, class, data_keys, input_keys. Error code: 2.
  2. An edge can only contain the following properties: label. Error code: 3.
  3. [Do not do for the moment] Duplicated properties in node or edge. Error code: 4.
  4. Duplicated state names, meaning there are at least two states with the same name (case sensitive). Error code: 5.
  5. Duplicated transitions, meaning there are at least two transitions with the same start and end states. Error code: 6.
  6. The value of the ‘start’ property in state is not a boolean (case insensitive true or false). Error code: 7.
  7. The value of the ‘immediate’ property in state is not a boolean (case insensitive true or false). Error code: 8.
  8. The value of the ‘class’ property in state is not a string. Error code: 9.
  9. The value of the ‘data_keys’ property in state is not a comma-separated string. Error code: 10.
  10. The value of the ‘input_keys’ property in state is not a comma-separated string. Error code: 11.
  11. The value of the ‘label’ property in a transition is not a string. Error code: 12.
  12. The start and end state of a transition is the same, for example, node -> node. Error code: 13.
  13. Transition label does not have the required prefix. For a transition start -> end, where the start state has a predefined base class. The label of of this transition must have a prefix which is defined by the out_edges from that base class. Use the data from ‘label_prefixes’ in KnownBaseClasses.classes to do the checking. Error code: 14.
  14. The DOT specification is not a directional graph. This means the DOT specification string does not start with ‘digraph’. Error code: 15.
  15. There are more than one states marked as start state. Error code: 16.
  16. The DOT specification is not valid DOT format. Error code: 17.
  17. A state inheriting from pre-defined states must have all the out-going transitions defined by the pre-defined state. For example, CancelOrderState2 has two out-going edges, whose labels should have prefix “fully traded” and “canceled”, so if state x inherits from CancelOrderState2, then there should have two edges x -> y[label=fully traded_SOME_OTHER_TEXT], and x -> z[label=canceled_SOME_OTHER_TEXT]. Error code: 18
