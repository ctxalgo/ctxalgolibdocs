bar_slice_in_days(ohlc, now=None, instrument_id=None, start=-1, end=None, is_trading_day=True)¶

Return a slice object indexing into bars of the given ohlc. The slice object covers the bars that are within the trading days defined by start and end.

Note: a day/trading day is counted if there is a timestamp (either from ohlc or now) within that day
is observed. For example, if now is Monday, and we are counting calendar days, Saturday and Sunday will not be counted, because there is no trading data timestamp within weekends.
For example,
bar_slice_in_days(ohlc): gives the bar range for trading days [-1:None], this is the last trading day
from ohlc.
bar_slice_in_days(ohlc, now): gives the bar range for trading days [-1:None], but makes use of
the now parameter. now can be different from the last timestamp in ohlc.
bar_slice_in_days(ohlc, start=-4, end=-2, is_trading_days=False): gives the bar range from calendar
day [-4:-2].
Parameters:
  • ohlc – AbstractOhlc, the ohlc object.
  • now – datetime, current time stamp. if None, defaults to the last timestamp inside ohlc.
  • instrument_id – string, the instrument id that ohlc belongs to. If None, use ohlc.id.
  • start – int, the calendar day or trading day start index. If None, defaults to end - 1. Must be negative int or None.
  • end – int, the calendar day or trading day end index. Must be negative int or None.
  • is_trading_day – boolean, if True, bars in a day are organized by trading day borders, otherwise, by calendar day borders.
:return slice, the Python slice object, which is used to index into ohlc to get the range of bars.
If there is no bars found in the specified day ranges, return None. If the given ohlc only contains part of asked days, return the slice for that parts.
trading_day_start_timestamps(instrument_id, start_ts, end_ts)¶

Return the start timestamps of every trading day between start_ts and end_ts ends. :param instrument_id: string, instrument id. :param start_ts: datetime, start timestamp, inclusive. :param end_ts: datetime, end timestamp, inclusive. An end timestamp after 17:00:00 will be

considered to belong to the next trading day. This is a simplification for the Chinese stock and future market.
Returns:[datetime], the list trading day start timestamp.
