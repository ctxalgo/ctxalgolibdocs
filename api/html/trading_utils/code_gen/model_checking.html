class ctxalgolib.trading_utils.code_gen.model_checking.ModelPostconditionTypeChecker¶

Bases: ctxalgolib.trading_utils.code_gen.model_checking.AbstractModelTypeChecker

Class that checks that model postconditions has the following structure: To get model postconditions from a state machine. Use the model_postconditions method from the state machine.

Possible errors: 1. Its predicate is an implication: implies(p, q).

Error code: 1.
  1. The implication condition p must have the structure: self.next_state() == ‘some_state_name’. Note single quotes. Error code: 2.
  2. The implication consequence q must have the structure: mv == some_expression. Error code: 3.
  3. And mv must be equal to a model variable. Error code: 4.
  4. And all accesses (attribute access, calls), except for old(...) and implies(...) in some_expression must be equal to a model variable as well. For example, if a model postcondition’s some_expression is “len(self.positions()) == 1” and “len(self.positions())” is not a model variable, then current type checker should flag an error. Error code: 5.
  5. The implication condition p: self.next_state() == ‘some_state_name’. The some_state_name must be a defined name from the state_machine.states. Error code: 6.

Class that type checks model state transition constraints. To get model state transition constraints, use the model_state_transition_constraints attribute from the state machine.

Possible errors for a constraint (start_state, end_state, label, constraint, line_number): 1. start_state is not defined in state machine.

Error code: 6.
  1. end_state is not defined in state machine. Error code: 6.
  2. constraint uses functions old(...). Error code: 61.
  3. constraint uses LTL functions F(...), G(...), U(...), X(...). Error code: 62.
  4. All accesses (attribute access, calls) must be model variables. For example, if a model postcondition’s some_expression is “len(self.positions()) == 1” and “len(self.positions())” is not a model variable, then current type checker should flag an error. Error code: 63.
  5. There is no transition from the start and end state, given by the constraint, in the state machine. For example, if the constraint says the start and end state is x -> y. But there is no transition from x to y Error code: 64.

Class that takes a state machine and generate Promela code for model checking. The code generationg consists of the following steps: * Take the original state machine, and use get_canonical_state_machine to

turn all expressions into canonical form.
  • Generate Promela variable names for all the model variables using generate_promela_variable_names.
  • Decide on the minimum number of temporary variables and their types: for a model postcondition in source_state: implies(self.next_state() == ‘target_state’, mv == some_expression) all mentions to model variables in some_expression will need to have a corresponding temporary variable.

Initialize current. :param state_machine: StateMachineInfo, information about the state machine.

static translate_ltl_property(expr)¶

Translate a Linear Temporal Logic predicate written in Python to its Promela syntax. Things to translate: 1. Translate G(p) into [] (p). 2. Translate F(p) into <> (p). 3. Translate U(p, q) into (p) U (q). 4. Translate implies(p, q) into (p) -> (q). 5. Translate logical a and b into a && b. 6. Translate logical a or b into a || b. 7. Translate logical not a into !a. :param expr: Expr, the Linear Temporal Logic predicate in Python syntax. :return: string, the translated property.
