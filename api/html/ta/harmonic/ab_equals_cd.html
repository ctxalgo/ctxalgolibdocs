ab_equals_cd¶

class ctxalgolib.ta.harmonic.ab_equals_cd.ABEqualsCDPattern(trading_direction, pivots, is_d_predicted, is_validated, zone, concentrated_areas)¶

Bases: ctxalgolib.ta.harmonic.harmonic_pattern.AbstractHarmonicPattern

Initialize current AB=CD pattern. Arguments:

trading_direction: TradingDirection, the trading direction of current pattern. pivots: [(int, float)], the list of 4 points, A, B, C and D.

Every point consists of two fields, the first int is the 0-based bar index, the second float is the price at that point. For A, the high prices, B, the low price, C the high price and D, the estimated or existing low price. The position of D is defined as follows: the bar index is the index which satisfies the high_A-low_B = high_C-low_D if D.

is_d_predicted: bool, True if D is a predicated value (it has not appeared yet), False otherwise. is_validated: bool, True if D is a validated value, False otherwise. zone: [(int, float)], the list of 2 points defining the top-left and bottom-right corner of the operation

zone (the blue zone).
concentrated_areas: [[up_ratio, up_value, up_ratio_source, down_ratio, down_value, down_ratio_source]]
The list of concentrated areas in the operation zone. see ConcentratedAreaFinder for the definition of the elements. In this case, AB related ratios are the first ratio list, BC related ratios are the second ratio list.
