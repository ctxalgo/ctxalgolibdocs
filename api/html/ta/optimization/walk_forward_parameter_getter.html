walk_forward_parameter_getter¶

class ctxalgolib.ta.optimization.walk_forward_parameter_getter.WalkForwardParameterGetter(initial_parameters, parameters)¶

Bases: object

Initialize current. :param initial_parameters: dict{string: object}, initial parameters.

Keys are parameter names, values are parameter values.
Parameters:parameters – Parameters used in different periods. Each element represents a period (a number of consecutive trading days) in which the strategy uses the same set of parameters. Each element is a dict {string: object}. The dict has the following keys: start_date, end_date and parameters.
parameters(trading_day)¶

Get the parameters used on the given trading day. Implementation note: this method should have constant execution time. This means no linear search based on trading_day. Use dict to find out the result. This means we need to store all dates between the walk-forwarded period. :param trading_day: date or string in form of yyyy-mm-dd, trading day. :return: dict{string: object} The parameters to use on trading_day, according to

the result of walk-forward optimization. Keys are parameter names, values are parameter values. For parameters participated in optimization, their values are decided by the walk-forward optimization. For parameters not participated in optimization or trading_day is before the first predication window, use self.initial_parameters to find the values for parameters.
