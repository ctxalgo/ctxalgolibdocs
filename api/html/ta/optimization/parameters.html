class ctxalgolib.ta.optimization.parameters.Parameter(name, name_mapping=None, is_numeric=False, init_value=None, transform=<function identity>, reverse_transform=<function identity>)¶

Bases: object

Specification of a parameter for optimization.

The interval value representaion is always a number no matter what the original domain of the parameter is. If the parameter is discrete, the value is the index of the value list. If the parameter is continuous, the value is the original continuous value.

If the parameter is used in a parameter dictionary (param_dict) in an objective, parameter name_mapping string provides a convenient (optional) way to specify the location of the parameter in param_dict. Mapping rule:

‘a:1:b,c:d’ correponds to two tied parameters: param_dict[‘a’][1][‘b’], param_dict[‘c’][‘d’]

The string is separated by ‘,’ where each element corresponds to one parameter in param_dict and each element is furthur separated by ‘:’ where each element can be an index or a string started with a non-digit character, indicating the key in one level of the param_dict.

Parameters:
  • name – parameter name.
  • name_mapping – a string corresponding to (possibly tied) parameters in the objective function. Use name instead if not provided.
  • is_numeric – true if the value can be treated as numeric. This can be useful when an optimizer wants to use the parameter value in the original domain whenever possible.
  • init_value – initial value in the original domain.
  • transform – transformation from the internal to the original domain.
  • reverse_transform – transformation from the original to the internal domain.
