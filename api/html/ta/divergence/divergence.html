divergence¶

class ctxalgolib.ta.divergence.divergence.Divergence(kind, price1, price_bar1, price2, price_bar2, momentum1, momentum_bar1, momentum2, momentum_bar2, confirm_bar, confirm_price=None, div_type='mu')¶

Initialize current divergence. A divergence is defined by four points: two points in the price series, and two points in the momentum indicator series. (price_bar1, price1) defines the first point in the price series, price_bar1 is the 0-based bar index in the price series, and price1 is the price value. (price_bar2, price2) defines the second point. Similarly, (momentum_bar1, momentum1) and (momentum_bar2, momentum2) defines the two points in the momentum indicator series. You can take a look at http://ctxalgo.com/pool?pool=x&sid=sh000001&indicators=macd(div=mu) to identify those points in detected divergences.

Parameters:
  • kind – int, 0 means top divergence; 1 means bottom divergence.
  • price2 (price1,) – float, value of prices. 6 & 8
  • price_bar2 (price_bar1,) – int, 0-based bar index of price1 and price2. 6 & 8
  • momentum2 (momentum1,) – float, value of the momentum indicator. 2 & 4
  • momentum_bar2 (momentum_bar1,) – int, 0-based bar index of momentum1 and momentum2. 2 & 4
  • confirm_bar – int, the 0-based bar index where the divergence is confirmed. None value means that current divergence is not confirmed.
  • confirm_price – float, the price used to confirm the divergence. If the divergence is not confirmed, set it to None.
  • div_type – string, type of the divergence, can be ‘mu’ or ‘bull_bear’.
start_bar()¶

Return the start bar of current divergence. Start bar is the smallest bar between price_bar1, price_bar2, momentum_bar1 and momentum_bar2. :return: int, 0-based bar index of the ohlc from which the divergence is found.

end_bar()¶

Return the end bar of current divergence. Start bar is the largest bar between price_bar1, price_bar2, momentum_bar1 and momentum_bar2. :return: int, 0-based bar index of the ohlc from which the divergence is found.

is_same_position(other)¶

If current divergence at the same position as the other divergence? :param other: Divergence, the other divergence. :return: boolean, True if the two divergences are at the same position.

class ctxalgolib.ta.divergence.divergence.AbstractDivergenceFinder¶
find(ohlc, momentum_indicator, kind=-1, max_count=-1, should_confirm=True, peeked_last_bar=None, div_type='mu')¶

Find divergences from ohlc and momentum indicator. :param ohlc: OHLC, ohlc time series. :param momentum_indicator: [float], momentum indicator. :param kind: int, 0 means to find top divergences; 1 means to find bottom divergences. -1 means find both. :param max_count: int, the maximal number of end points where some divergence ends to find.

-1 means find all.
Parameters:
  • should_confirm – boolean, If True, the returned divergences must be confirmed. If False, the returned divergences can be or can not be confirmed. If a divergence is not confirmed, it’s confirm_bar is set to None.
  • peeked_last_bar – Bar, the bar which has not fully developed, used in confirmation. It can be None if no partial bar information is available. If should_confirm is False, peeked_last_bar is not used by find. div_type: string, The type of the divergence. Values can be: ‘mu’: Find the MuYang type divergences. ‘bull_bear’: find bullish and bearish divergences.

:return [Divergence] The list of found divergences.

confirm(divergence, ohlc, momentum_indicator, peeked_last_bar=None)¶

Confirm divergence. If divergence is confirmed, set its confirm_bar to the 0-based bar index where it is confirmed. If the confirm is done inside peeked_last_bar, then set confirm_bar to -1. If the divergence is not confirmed. Set confirm_bar to None. :param divergence: Divergence, the divergence to be confirmed. :param ohlc: OHLC, the OHLC object. :param momentum_indicator: [float], the values of a momentum indicator. :param peeked_last_bar: Bar, the peeked last bar if not None.

Peeked last bar is a bar which has not fully developed. Note that for the peeked_last_bar, there is no corresponding value in ohlc and momentum_indicator, because those values are only generated when a full bar has developed. So if the confirmation needs to use peeked_last_bar, it will assume that the value of the momentum indicator is the same as the last value in momentum_indicator.
Returns:this method does not return any value.
