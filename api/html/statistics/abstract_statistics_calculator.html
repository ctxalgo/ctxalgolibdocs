start_output_date()¶

Return the trading day starting from which add_daily_data starts to output non empty dict result. :return: date. None value means add_daily_data should always output result when available.

add_daily_data(trading_day, data)¶

Add daily data on the given trading day into current calculator. :param trading_day: date, current trading day. :param data: dict{string: object}, the data on the trading day.

The actual content in data is flexible and only can be defined in descendant classes.
Returns:dict{date: dict{string: object}}, meaning dict{trading_day: dict{statistics_name: value}}. The outer keys are trading days, this means that one new trading day’s data may trigger the generation of statistics from more than one (previous) trading days. The inner keys are statistics names, one statistics calculator may output multiple statistics. The inner values are the the actual statistics values on the corresponding trading days. Notes: 1. 1f on a trading day, there should be no statistics to calculate, for example, if a statistics
needs 3 days of trading data, but we only have 2 days’ of data, then do not include the trading day as key in the result dict.
  1. If on a trading day, some statistics calculated as np.nan, for example, if the statistics
    is a mean but we don’t have any element to average, then set the statistics to np.nan.
