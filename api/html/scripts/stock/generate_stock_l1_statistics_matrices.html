generate_stock_l1_statistics_matrices¶

The script has the following command line options:

How the matrices are generated: 1. Decide the matrix size

Given START_DATE, END_DATE and DATE_INDEX, find start date and end date in the date index, and calculate the number of trading days between start_date and end_date (both inclusive), let the number of trading days DAYS be the number of rows in the matrix.

Given the STOCK_INDEX, the number of stocks STOCKS in that file is the number of columns of the matrix.

So the size of the matrix is: DAYS x STOCKS.

  1. Initialize each matrices to an numpy ndarray with all numpy.nan values. For each statistics, we generate a matrix.

    Every matrix is of size DAYS x STOCKS.

  2. Sort the files in INPUT_FOLDER by their names ascendingly, and read those sorted files, one row at a time.

    Each row has the format: trading_day, stock, statistics_name, value

    Use the trading_day to decide the row number to put into the matrix. Note the row in the matrix is index(trading_day) - index(START_DATE), where index is the function that returns an integer index from a trading day, which is defined by DATE_INDEX.

    Ue the stock to decide the column number to put into the matrix.

    Use the statistics_name to decide which matrix to put the value in.

    Put the value into the correct position into the correct matrix. Note that the same (trading_day, stock, statistics_name, value) tuple may appear multiple times when there are multiple files in INPUT_FOLDER, always use the last None non value as the value to put into the matrix. Meaning that If on a trading day, if the statistics of a stock is None, then no need to update the matrix.

    Note: if a stock is not in STOCK_INDEX or a date is not in DATE_INDEX, ignore that tuple.

  3. When all the files from INPUT_FOLDER have been exhausted, output the matrices into OUTPUT_FOLDER.
