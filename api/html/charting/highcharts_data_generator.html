Bases: ctxalgolib.charting.chart_data_generator.AbstractChartDataGenerator

Class to generate JSON data for chart display.

Initialize current. :param translator: AbstractTranslator, a translator to provide translations into different languages. :param data_feed: AbstractDataFeed, data feed to provide ohlc and other such as stock fundamental data. :param instrument_id: string, the id of the instrument to draw. :param periodicity: Periodicity or int/float, the period for the ohlc to draw.

In case of time-based ohlcs, the value is Periodicity, in case of volatility-based ohlcs, the value is volatility threshold.
Parameters:
  • ohlc_kind – string, can be ‘time-based’, ‘volatility-based’ from OhlcGeneratorConstants. It specifies the kind of ohlc to draw.
  • start_time – datetime, the start time (inclusive) to draw the ohlc.
  • end_time – datetime, the end time (inclusive) to draw the ohlc.
  • limit_bars – boolean, if True, decide if there are two many ohlc bars to draw, if so, only draw a postfix of the required ohlc bars.
  • ohlc_attributes –

    dict{string: value}, indicates which additional fields should be included in ohlc data. keys are open_interests, dominants, profits, order_book_level. If None, default is: {

    open_interests: False, dominants: False, profits: False, order_book_level: 0

    }

generate_html(folder, instrument_id=None, cursor_date=None, indicators=None, switch=None, description=None, mobile=False, limit_bars=False, html_file='index.html', include_all_files=False, js_file_location=None)¶

Generate a static html page that contains chart(s) for instrument_id and indicators. If the indicators parameter specifies multiple charts via indicators1= to indicators5, the generated html page should contains all those sub-charts. The html page may require javascript files, get those files from the html folder in current Python package.

The static html page should:
  1. Open all the specified charts.
  2. No need to handle any web page widget actions. For example, no page reload, no switch on/off sub-charts (you can disable the switches).
  3. Need to include title and sub-title (if any).
Parameters:
  • folder – string, the folder to store the generated web page. If folder does not exist, create it. If it exists, keep the existing files there. New files should be generated and override existing files, if any.
  • instrument_id – string, the instrument id. If you don’t want to plot data related to any instrument, set instrument_id to None.
  • cursor_date – date, the cursor date.
  • indicators – [string], The list of indicators, the first element is for the main chart, the second element is for the first sub-chart and so on.
  • switch – int, indicates how many charts to open. If None, use the length of indicators to calculate the actual switch value.
  • description – string, the pattern specification, used to draw a flag pattern.
  • mobile – boolean, if True, generate chart data for mobile device. Otherwise, for normal screen.
  • limit_bars – boolean, if True, limit the maximum number of bars generated for charting.
  • html_file – string, the name of the generated html file.
  • include_all_files – boolean, if True, include all css, js files under folder, otherwise reference those files from ctxalgolib, or in case that js_file_location is not None, from the location given by js_file_location.

:param string, provide the location to reference js files.
