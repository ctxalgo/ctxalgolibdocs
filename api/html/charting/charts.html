charts¶

class ctxalgolib.charting.charts.Charts(data_feed=None, translator=None, folder=None, include_all_files=False, open_in_browser=True, ohlc_attributes=None, period=None, volatility_threshold=None)¶

Bases: object

Class that generates html-based charts.

Initialize current. :param data_feed: AbstractDataFeed, BacktestingDataSource or string,

the data feed used by current to load trading data. If it is AbstractDataFeed, then use the data feed directly. If it is a BacktestingDataSource, a data feed will be constructed based on that data source. If it is a string, the string represents a path to a data file or a folder containing data files.

A FileBasedDataFeed will be crated internally.
If it is a OHLC, create a OhlcBasedDataFeed object for the ohlc object. And you have to specify either
period or volatility_threshold.
Parameters:
  • translator – AbstractTranslator, the translator, used in i18n.
  • folder – string, the folder to store all generated html files. If None, a temp folder will be created. After use, you need to call the dispose method to r emove the generated html files and in case a temp folder is created, delete that temp folder.
  • include_all_files – boolean, if True, generate all files in folder, including javascript files. If False, only generate the html page, and the page will reference javascript files from ctxalgolib.
  • open_in_browser – boolean, if True, calling the show method will open the generated html page in browser, if False, calling the show method only returns the path of the generated html page.
  • ohlc_attributes –

    dict{string: value}, indicates which additional fields should be included in ohlc data. keys are open_interests, dominants, profits, order_book_level. If None, default is: {

    open_interests: False, dominants: False, profits: False, order_book_level: 0

    }

  • period – Periodicity, only used to specify the time-based period if data_feed is an OHLC object.
  • volatility_threshold – int/float, only used to specify the volatility-based threshold if data_feed is an OHLC object.
figure()¶

Clear all constructed chart information and start afresh.

set_instrument(instrument_id)¶

Set the default instrument id with instrument_id. If the chart only involves a single instrument, set that instrument’s id here can simplify chart construction, because you don’t need to specify instrument ids the individual builder methods, such as self.series. If the chart involves multiple instruments, you can leave this instrument id None and specify instrument ids in all builder methods, or set it to a default instrument id, and specify other instrument ids in building methods which charts for other instruments. :param instrument_id: string, instrument id.

sub_chart(index)¶

Specify which sub-chart to accept chart construction information. :param index: int, 0-based chart index. 0 means main chart.

show(html_file=None, should_dispose_html_file=False)¶

Generate html page representing the current chart construction and return the html page path. If self.open_in_browser is True, also try to open the generated html page in browser. :param html_file: string, the path to a file which will be the generated html file in self.folder.

If None, create a file inside self.folder.
Parameters:should_dispose_html_file – boolean, only have effect when html_file is not None. If True, the generate html_file will not be deleted when self.dispose is called.
Returns:string, the path to the generated html page.
dispose()¶

Remove all generated files and temp folders. Should be invoked when you are done with the charting facility.

Plot the balance time series from trading strategy report. :param period: Periodicity or string, the period to plot the balance series.

If None, use default period.
Parameters:
  • balance – OHLC, the balance series. The dates and closes attributes in the ohlc represents the balance series. If not None, the data in this ohlc will be used, and report will be ignored.

  • type – string, the line type of the balance series, valid values are line and step.
  • style – string, the dash style, valid values are: Solid, ShortDash, ShortDot, ShortDashDot, ShortDashDotDot, Dot, Dash, LongDash, DashDot, LongDashDot, LongDashDotDot.
  • color – string, the color of the series.
  • thick – int, the width of the series.
  • height – int, the height of the chart, if None, a default will be used.
  • position – string, the position of the series.
  • opposite – boolean, if True, draw the curve on the opposite Y-aixs (on the right, instead of left).
  • net_value – boolean. If True, display balance as net values (starting from 1).
  • report – AbstractStrategyReport, the strategy backtesting report. If None, self.report is used.

instrument_positions(instrument_id=None, positions=None, period=None, merge=False, label=None, type='step', style='Solid', long_color='red', short_color='green', thick=2, report=None)¶

Draw instrument positions. :param instrument_id: string, instrument id. If None, use self.instrument_id. :param positions: dict{string: OHLC}, meaning dict{position_direction: OHLC}. It can also

be of type dict{int: OHLC}, where key of 1 represents long and key of -1 represents short. Keys are position directions, can be ‘long’ or ‘short’. Values are ohlcs indicating the positions. The dates and closes attributes from those ohlcs presents position series. If positions is not None, its data will be used, and report will be ignored. If you only have positions from one direction, you don’t need to provide the series for the other direction.
Parameters:
  • period – Periodicity or string, the period to plot the balance series. If None, use default period.
  • merge – boolean, if False, do not perform period-based merging. Leave it as False if you want the plotted positions correspond to the transaction start and end points (see self.transaction).
  • label – string, the label of the instrument positions area.
  • type – string, the line type of the balance series, valid values are line and step.
  • style – string, the dash style, valid values are: Solid, ShortDash, ShortDot, ShortDashDot, ShortDashDotDot, Dot, Dash, LongDash, DashDot, LongDashDot, LongDashDotDot.
  • long_color – string, color for long positions.
  • short_color – string, color for short positions.
  • thick – int, the thickness of the plotted series.
  • report – AbstractStrategyReport, the strategy backtesting report. If None, self.report is used.
transactions(instrument_id=None, transactions=None, direction=None, start_time=None, end_time=None, long_color='red', short_color='green', position='k', pnl=False, report=None)¶

Draw individual transactions. :param instrument_id: string, instrument id. If None, use self.instrument_id. :param transactions: [ctxalgolib.report.abstract_strategy_report.AbstractTransaction],

the list of transactions to plot. If None, use all transactions from self.report.
Parameters:
  • direction – string, ‘long’ or ‘short’, plot only transactions in the given direction. If None, plot transactions in both directions.
  • start_time – datetime, If None, use the first available time.
  • end_time – datetime, If None, use the last available time. start_time and end_time define a time range, only transactions within this range are plotted.
  • long_color – string, color for long positions.
  • short_color – string, color for short positions.
  • position – string, the position of the series.
  • pnl – boolean, if True, plot the profit and loss of the transactions.
  • report – AbstractStrategyReport, the strategy backtesting report. If None, self.report is used.
