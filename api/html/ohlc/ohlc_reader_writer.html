class ctxalgolib.ohlc.ohlc_reader_writer.OhlcCsvReader(instrument_id, path=None, with_open_interests=False, order_book_level=0, dominant=False, profits=False, time_precision=0, header=False, line_parser=None)¶

Bases: ctxalgolib.ohlc.ohlc_reader_writer.AbstractOhlcReader

Initialize current. :param instrument_id: string, the instrument id of the ohlc to be read. :param path: string, the file path to read from. :param with_open_interests: boolean, if True, include the open_interests data in the output. :param order_book_level: int, 0 means no order book is included in the output.

A positive integer means that include that level and below order books.
Parameters:
  • dominant – boolean, if True include the dominant column.
  • profits – boolean, if True include the profits column.
  • time_precision – int, 0 means second precision, 1 means millisecond precision. This means, that the read datetime should have second or millisecond precision.
  • header – boolean, if True, the underlying csv data file contains a header line, so ignore that header line when reading data.
  • line_parser – a function pointer which takes a row of data as string, and return the fields that are needed for a ohlc bar. The actual fields needs to be the same as self.required_attributes. The return value is of type dict{string: object}, the keys are attribute names. If not None, use this parser to parse the data file.

read(path=None)¶

Use BinaryFileTimeSeries to do the reading. The dates and timestamps attributes in ohlc should be written as datetime64 with proper time precision (see self.time_precision), the other attributes in ohlc should be written as float64. After reading, covert all numpy objects into native Python object. This means numpy.datetime64 -> datetime, numpy.float64 to float.
