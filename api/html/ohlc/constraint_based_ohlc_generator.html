class ctxalgolib.ohlc.constraint_based_ohlc_generator.RelativeOhlcBarConstraint(source_date, source_dimension, c, operator, target_dimension, target_start_date=None, target_end_date=None)¶

Bases: ctxalgolib.ohlc.constraint_based_ohlc_generator.AbstractOhlcBarConstraint

Relative constraint describes the property of a dimension (called target dimension) of a set of rows (called target rows) with respect to a dimension (called source dimension) of another row (called source row). For example, high value of a set of rows must be smaller than the low value of another row.

Example constraints: 1. high of row2 and row3 is larger than close of row1 + 5.

target_dimension=high, target_start_date and target_end_date=row2, row3, source_date=row1, source_dimension=close, operator is >, c=5.
  1. close of row10 to row20 equals the low of row2. target_dimension=close, target_start_date and target_end_date=row10,row20, source_date=row2, source_dimension=low, operator is =, c=0.
Parameters:
  • source_date – datetime, the timestamp of the source row. Note, the source is always a single row.
  • source_dimension – string, the dimension name of the source row.
  • c – int/float, a constant.
  • operator – string, the constraint operator.
  • target_start_date – datetime, the stat timestamp of target rows, inclusive. If None, use the first timestamp in the ohlc.
  • target_end_date – datetime, the end timestamp of target rows, inclusive. If None, use the last timestamp in the ohlc.

class ctxalgolib.ohlc.constraint_based_ohlc_generator.ConstraintBasedOhlcGenerator(original_ohlc)¶

Bases: object

This class will generate a new ohlc object based on a given original ohlc object and a set of constraints. The generated new ohlc object should be similar to the original ohlc object but with the set of constraints met.

Example 1 We want a new ohlc, whose close price from 09:20 to 09:30 is set to 100, do:

g = ConstraintBasedOhlcGenerator(original_ohlc) g.add_constraint(OhlcAttributes.closes, ‘=’, 100, start_date=09:20:00, end_date=09:30:00) new_ohlc = g.generate()

Example 2 We want a new ohlc, whose high prices between 09:20 and 09:30 must > 100 and low prices must < 50, do:

g = ConstraintBasedOhlcGenerator(original_ohlc) g.add_constraint(OhlcAttributes.highs, ‘>’, 100, start_date=09:20:00, end_date=09:30:00) g.add_constraint(OhlcAttributes.lows, ‘<’, 50, start_date=09:20:00, end_date=09:30:00) new_ohlc = g.generate()

Intrinsic constraints on every row in an ohlc object: 1. highs is the highest value over opens, highs, lows, closes: highs = max(opens, highs, lows, closes) 2. lows is the lowest value over opens, highs, lows, closes: lows = min(opens, highs, lows, closes) 3. dimensions is non-negative: forall v in self.supported_dimensions => v >= 0

Implementation ideas: We can use a SAT solver (the most powerful one being Z3 https://github.com/Z3Prover/z3). For each dimension of each relevant rows (not all rows need change) in the original ohlc, introduce a variable. For example, the close price of rows will be c1, c2, c3, ..., cn where 1, 2, 3, n are row indices, and the volume of rows will be v1, v2, ..., vn. And we use c1’ to represent the new value of c1 after the constraints are met. For example.

The we can translate the constraints: c1’ = 100, the close price is set to 100. c1’ <= h1, the close price must be smaller than or equal to the high price (intrinsic constraint). c1’ >= l1, the close price must be larger than or equal to the high price (intrinsic constraint).

To model the similarity of the new ohlc to the original ohlc, introduce the following minimization objective:

Minimize: (c1’-c1)^2 + (c2’-c2)^2 + (cn’-cn)^2 + ... + (v1’-v1)^2 + ...

To use the class, you must have z3 installed, follow instructions from https://github.com/Z3Prover/z3 to install z3. After installation, you need to setup z3. Follow the instructions from the z3 python build example file, located in z3/build/python/example.py to setup. And you need to declare an environment variable Z3_LIBRARY_PATH pointing to the z3 dll/so file, located in z3/build/python.

Parameters:
  • original_ohlc – OHLC, the ohlc object, on which constraint-based generation is based.
  • z3lib_path – string, the path to the libz3.dll.
