Return an ohlc generator of the given specs. :param kind: string, the kind of ohlcs, such as ‘time-based’, ‘volatility-based’. :param market_periods: [(time, time)], a list of time periods defining the open periods (in one day)

for the market. Each element of the array defines a period. The first element is the open time, the second element is the close time. The open time of the first period and the close time of the last period defines the market open and close time for a day. market close time can be smaller than market open time, this means that the market opens at the previous date. For example, for 黄金期货, market open time = time(21, 0, 0) and market close time = time(15, 0, 0). This means for that future, a day’s trading starts at date1 21:00, and ends at date1+1 15:00. This affects the calculation of daily bars.
Parameters:
  • ohlcs – dict from Periodicity or int/float to OHLC. Keys are periodicities of the ohlcs to generate in the case of time-based ohlcs, or volatility thresholds in the case of volatility-based ohlcs. Values are ohlcs which serve as existing data. Newly generated bars will be appended to those ohlcs. If an ohlc is None, it will be initialized to an empty ohlc.
  • on_bar_generated –

    a function pointer, which is invoked when one or more bars (with different periodicities) are fully generated. The function takes three parameters. 1. The type of the first parameter is string, it tells the kind of the ohlc,

    such as ‘time-based’ or ‘volatility-based’.
    1. The type of the second parameter is dict from Periodicity to Bar, representing the bars that are generated.
    2. The type of the third parameter is arbitrary, it is the context data that has been passed through from on_market_data.
  • offset – Not used.
  • order_book_level – int, 0 means no order book information will be included in the generated ohlc data. A positive number means 1~that number of order book levels information will be included in the generated ohlc data. For example if order_book_level=3, then ask_price1~ask_price3, ask_volume1~ask_volume3, bid_price1~bid_price3, bid_volume1~bid_volume3 and last_price, last_volume and timestamp will be included in every row of the generated ohlc.
  • trading_days – TradingDays, the trading days object.
  • is_source_data_tick – boolean, indicates whether the source data is tick level.
Pram with_open_interest:
 

bool, if True, the data from which ohlc bars are generated com with open interest. Only suitable for future data.

on_market_data(dt_, open_, high_, low_, close_, volume_, context, open_interest=None)¶

An action to be invoked when some market trading data arrives. This method will use the arrived data to generate ohlc bars. All generated ohlc bars will be appended to self.ohlcs. Note:

  1. The market information in the parameters of this method is used to generate bars if the time stamps within self.market_periods (open time exclusive, close time inclusive).
  2. However, the information after the close time of a period is used to signal the finishing of the previous bar.
  3. When self.on_bar_generated is not None, invoke it if one or more bars are fully developed.
Parameters:
  • dt – datetime, the time stamp.
  • high, low, close, volume (open,) – float, the market trading data.
  • context – an arbitrary object serve as some data provider.
  • open_interest – float, the open interest.

:return dict{Periodicity:Bar}, the generated bars. Return an empty dict if there is no bar that is generated.

class ctxalgolib.ohlc.ohlc_generator.OhlcGenerator(ohlcs, market_periods=None, on_bar_generated=None, offset=None, with_open_interest=False, order_book_level=0, dominant=False, profits=False, use_mid_ask_bid=False, trading_days=None, keep_new_bars=True, is_source_data_tick=False, profit_factor=None)¶

Bases: ctxalgolib.ohlc.ohlc_generator.AbstractOhlcGenerator

Initialize current. :param market_periods: [(time, time)], a list of time periods defining the open periods (in one day)

for the market. Each element of the array defines a period. The first element is the open time, the second element is the close time. The open time of the first period and the close time of the last period defines the market open and close time for a day. market close time can be smaller than market open time, this means that the market opens at the previous date. For example, for 黄金期货, market open time = time(21, 0, 0) and market close time = time(15, 0, 0). This means for that future, a day’s trading starts at date1 21:00, and ends at date1+1 15:00. This affects the calculation of daily bars.
Parameters:
  • ohlcs – dict from Periodicity to OHLC. Keys are periodicities of the ohlcs to generate, values are ohlcs which serve as existing data. Newly generated bars will be appended to those ohlcs. If an ohlc is None, it will be initialized to an empty ohlc.
  • on_bar_generated – a function pointer, which is invoked when one or more bars (with different periodicities) are fully generated. The function takes three parameters. The type of the first parameter is string, it tells the kind of the ohlc, such as ‘time-based’ or ‘volatility-based’. The type of the second parameter is dict from Periodicity to Bar, representing the bars that are generated. The type of the third parameter is arbitrary, it is the context data that has been passed through from on_market_data.
  • offset – Not used.
  • order_book_level – int, 0 means no order book information will be included in the generated ohlc data. A positive number means 1~that number of order book levels information will be included in the generated ohlc data. For example if order_book_level=3, then ask_price1~ask_price3, ask_volume1~ask_volume3, bid_price1~bid_price3, bid_volume1~bid_volume3 and last_price, last_volume and timestamp will be included in every row of the generated ohlc.
  • dominant – boolean, if True, include the attribute for dominant contract switching.
  • use_mid_ask_bid – boolean, if True, ohlc bars are generated using mid ask-bid values.
  • keep_new_bars – boolean, if True, newly generated bars are added into the ohlcs objects kept in current object. If False, no new bars are kept.
  • is_source_data_tick – boolean, it indicates if the source information is tick data. If True, use the [) instead of (] semantics to decide which tick goes into which bar. the [) semantics means that the first tick whose timestamp is >= the end timestamp of a bar should trigger the bar generation. And that tick goes into the tick information of the generated bar, but the tick’s data is included in the next bar. For example, if we are generating one minute bars and a tick at 9:10:00.0 arrives, then the bar ending at 9:10 should be generated, its tick related field (in case order_book_level > 0) should be set with the tick, but the tick itself does not participate in calculating the open, high, low, close, volume of that bar.
  • profit_factor – float, the factor used to calculate the profit series.
Pram with_open_interest:
 

bool, if True, the data from which ohlc bars are generated com with open interest. Only suitable for future data.

on_market_data(dt_, open_, high_, low_, close_, volume_, context, open_interest=None)¶

An action to be invoked when some market trading data arrives. This method will use the arrived data to generate ohlc bars. All generated ohlc bars will be appended to self.ohlcs. Note:

  1. The market information in the parameters of this method is used to generate bars if the time stamps within self.market_periods (open time exclusive, close time inclusive).
  2. However, the information after the close time of a period is used to signal the finishing of the previous bar.
  3. When self.on_bar_generated is not None, invoke it if one or more bars are fully developed.
Parameters:
  • dt – datetime, the time stamp.
  • high, low, close, volume (open,) – float, the market trading data.
  • context – an arbitrary object serve as some data provider.
  • open_interest – float, the open interest.

:return dict{Periodicity:Bar}, the generated bars. Return an empty dict if there is no bar that is generated.
