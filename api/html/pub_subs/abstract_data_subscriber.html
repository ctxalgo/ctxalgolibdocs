receive()¶

Receive message. This is a blocking operation. So it should be used together with a poller. The poller first checks if there is data available for current subscriber, and if there is, then receive is called to make sure there is no blocking. :return: dict{string: object}, the received data. Keys are data names, values are the data items.

None if no data is received, or error occurs.
