tick_speed_rule¶

[en] Title: Tick arrival speed This rule checks if ticks arrives at at least a minimum speed (number of ticks/minute). The parameters of the rule:

```python {

‘sources’: [‘market_broker1’, ‘market_data_processor’], ‘speeds’: {

‘cu00’: 60, ‘rb00’: 70

}, ‘default_speed’: 1

}¶

The sources entry lists all the tick count sources that the rule is going to monitor. speeds defines the minimum number of ticks per minute for the given instruments during their market periods. default_speed defines the minimum number of ticks per minute for the instruments not specified in speeds.

[zh] Title: Tick到达的速度 这个规则报告Tick的到达速度是低于指定阈值。规则的参数如下：

```python {

‘sources’: [‘market_broker1’, ‘market_data_processor’], ‘speeds’: {

‘cu00’: 60, ‘rb00’: 70

}, ‘default_speed’: 1

}¶

`sources`列出了需要监听的tick源，可以列出多个源。`speeds`定义每个合约品种的最低tick速度，单位为tick个数/分钟。 `default_speed`指定那些没有在`speeds`中列出的合约的最低速度。

class ctxalgolib.rule_checking.rules.tick_speed_rule.TickSpeedRule(name, trading_day, parameters, message_validity_checker)¶

Bases: ctxalgolib.rule_checking.rule.AbstractRule

This rule checks if tick speed is above a given threshold. The input messages will be: TICK_COUNT#source {

‘tick_counts: {
‘cu00’: 5, ‘rb00’: 10,

}

}

TIMER

The tick_counts values, for example, ‘cu00’: 5, are non-decreasing. The values mean the number of ticks that have been received for those instruments on current trading day. Normally, there will be a TICK_COUNT message per source every 30 seconds.

The parameters of the current rule is:
parameters = {

‘sources’: [‘market_broker’,` ‘market_data_processor’], ‘speeds’: {

‘cu00’: 60, ‘rb00’: 70

}, ‘default_speed’: 1

}

The sources entry lists all the tick count sources that the rule is going to monitor. speeds defines the minimum number of ticks per minute for the given instruments during their market periods. default_speed defines the minimum number of ticks per minute for the instruments not specified in speeds.

Rule checking semantics: 1. When a TICK_COUNT message arrives, look up the market periods for the included instruments using

self._get_market_periods.
  1. When a TIMER message arrives, use the timestamp, along with instrument market periods, tick counts to decide if the rule should be violated for some instrument from some source.
  2. If the rule is violated for some instrument from some source, market this (instrument,source) pair as violated, and don’t trigger the violation again until the tick speed for this pair returns to above threshold.
  3. Only need to check during market periods and the checking starts from the first TICK_COUNT message we receive per source.
